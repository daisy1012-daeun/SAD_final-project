* 역할: 너는 웹 개발자이자 디자이너이다. 
* 목표: Supabase(Postgres) + Next.js(App Router) + Vercel 배포를 전제로, “전화번호 관리 시스템”의 PRD를 작성하라.
* 핵심 기능(필수)
    * 1. 지역 설정 : 학교 -> 단과대설정 -> 학과설정

2. 기존 생성형 AI와의 차별성을 위해 바코드 하나당 1포인트, 하루 최대 100포인트 마일리지 적립 방식으로 축척(추가 개발 가능할 시 학내 식당, 카페 등에서 사용 가능한 쿠폰 지급)

3. 체크 리스트 예시 : 라벨을 떼었나요?, 내용물을 씻었나요?, 뚜껑을 분리했나요? 등 제품 재질별 항목 제공

4. 음성 기능 추가 EX) 계란 껍질, 치킨 뼈 등 헷갈리는 품목 안내 (목소리랑 말 좀 예쁘게)

5. 친구 초대 / 같이하기 룸메이트나 친구를 초대하면 둘 다 보너스 포인트 500포인트.

6. 오늘의 분리배출 미션 "오늘은 플라스틱 3개 분리배출하기" 같은 챌린지, 달성하면 1,3,5,10 포인트 중  랜덤 포인트 증정.(+ 이건 하루 최대 100포인트와는 별개)

7. 암호화 로그인 기능

( 가능하면 단과대별 단과대 단위 경쟁으로 참여 유도 )



* 보안/암호화(필수)
    * DB에는 평문으로 이름/전화번호가 저장되면 안 된다.
    * 애플리케이션 레벨 암호화로 저장한다.
    * 암호화 키는 환경변수로 관리한다(Vercel 환경변수 + 로컬 .env).
    * 어떤 필드를 암호화할지, 검색이 어려워지는 트레이드오프를 PRD에 명시하라.
    * 검색을 위해 최소한의 보조 컬럼(예: 해시/인덱싱용)을 설계하고, 보안 상의 주의점을 PRD에 포함하라.
* 데이터 모델(필수)
    * contacts 테이블 스키마 초안을 제시하라.
    * created_at, updated_at 포함
    * 사용자 범위(개인용 단일 사용자 / 로그인 포함 멀티 사용자) 중 하나를 선택하고 이유 명시
* 기술 제약/선택(권장)
    * Next.js + TypeScript
    * Supabase JS 클라이언트
    * ORM은 선택(Prisma 또는 SQL 직접)
    * UI는 간단히(Shadcn/UI 또는 기본 컴포넌트)

    * 바코드 및 큐알 사진 찍기 위한 카메라 기능
    * 바코드/큐알 없는 제품을 위한 간단한 LLM 모델
    * 포인트 지급 모델
    * 로그인 기능(학교 한정 앱이기 때문에 학번과 연관될 수 있도록)
    * 목록 페이지(무한스크롤은 선택)
